## 🎨 設計の癖・ハマりどころ

### 税モード（v1.8.59〜）
- 設定で「本体／総額／ダッシュボード／展示元」をそれぞれ「税込／税抜」で独立切替可能
- 税率はデフォ10%、`appSettings.priceTax.rate` で変更可
- 表示時は `getTaxLabel(field)`、変換は `convertTax(amount, from, to)`
- **納車完了時に `priceTaxSnapshot` を保存**。後で設定変えても過去データは固定で集計される（`_amountFromCarWithSnapshot`）

### リアルタイム同期（v1.8.0〜）
- `onSnapshot` で他端末の変更を受信
- **編集中の車を上書きしないように `protectedIds` 機構あり**
  - `editingCarId` / `wsActiveCarId` / `activeDetailCarId` を main.js で管理
  - これを忘れると編集中にカード詳細が消えるバグ発生（v1.8.50で修正）

### 大タスクの選択制（v1.8.51〜）
- `appTaskOptional.regen[taskId]` / `delivery[taskId]` で「選択制」フラグ
- 各車両ごとに `car.optTasksRegen / optTasksDelivery` で opt-in
- 進捗は均等割り＋小タスク按分

### オーダー車両（v1.8.72〜）
- `car.isOrder = true` で表現
- **在庫扱いから除外**（在庫日数警告／ダッシュボード集計／商談ビューから除外）
- 展示ビューには右側に専用カラム「📦 オーダー車両」
- 売約は手動でトグル、契約済みになれば着地予測の「見込み」カウントに入る

### ロール
5段階: `admin / manager / staff / worker / viewer`
`js/role-guard.js` で MutationObserver による上書きあり（v1.8.10 で詰まった）

### キャッシュバスター
全 script/link タグに `?v=NNN` 付きで管理。**index.htmlの数字を都度bumpする**こと（強制リロード対策）。

## 🚦 デプロイ作業の鉄則

1. `index.html` のバージョン表示（`login-ver` / topbar）2ヶ所を bump
2. 変更した js/css の `?v=` を bump（複数ある場合は全部）
3. `cd /d D:\アプリ開発\CarFlow\carflow && firebase deploy --only hosting`
4. ブラウザで Ctrl+Shift+R で確認

**WARNING:**
- `D:\アプリ開発\CarFlow` ではなく `D:\アプリ開発\CarFlow\carflow` から実行（firebase.json は後者にある）
- `cd D:` だけだと drive 移動できないので `cd /d` を付ける
